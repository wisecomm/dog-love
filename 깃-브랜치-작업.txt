

PR 만드는 순서 (터미널 명령어)
1. 새 브랜치 만들기 (예: feat/login 기능 개발)
git checkout -b feat/login  # 브랜치 생성 및 이동

--- 서버 동기화 
git pull --rebase origin main 
수정 시 : git add + git commit.


2. 작업 후 커밋하기
# (코드 수정 후)
git add .
git commit -m "feat: 로그인 기능 추가"
gh pr create --fill  # push + PR 생성을 한 번에! (제목은 커밋에서 자동)


############################
"아예 없던 일로 할래!" (삭제)
1. 일단 밖으로 나옵니다
git checkout main
2. 브랜치를 강제 삭제합니다
git branch -D feat/login
