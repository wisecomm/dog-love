PR URL: $@

각 PR에 대해 다음 순서로 수행:

1. PR 페이지 전체를 읽는다 (설명, 모든 댓글, 커밋, 변경 파일).
2. PR 본문, 댓글, 커밋 메시지에서 연결된 이슈를 찾아 전체를 읽는다.
3. PR diff를 분석한다. main 브랜치의 관련 코드 파일을 전체 읽고 diff와 비교한다. diff에 포함되지 않았지만 동작 검증에 필요한 관련 코드 경로도 확인한다.
4. 다음 구조로 리뷰를 작성한다:

## 출력 형식

```
PR: <url>

Good:
- 좋은 설계 결정이나 개선사항

Bad:
- 구체적 문제, 리그레션, 누락된 테스트, 리스크

Ugly:
- 미묘하거나 영향이 큰 문제

질문/가정:
- 불명확한 사항

변경 요약:
- 변경 내용 정리

테스트:
- 테스트 현황 및 필요한 테스트
```

문제가 없으면 Bad, Ugly에 "없음"으로 표시.

## 검증 기준

- TypeScript 타입 안전성
- React Hook 규칙 준수
- XSS, injection 등 보안 취약점
- React Query 캐시 무효화 누락
- Zod 스키마 검증 누락
- 불필요한 리렌더링 유발 코드
