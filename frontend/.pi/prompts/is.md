GitHub 이슈 분석: $ARGUMENTS

각 이슈에 대해:

1. 이슈 전체를 읽는다 (모든 댓글, 연결된 이슈/PR 포함).

2. **버그인 경우**:
   - 이슈에 작성된 원인 분석은 무시한다 (대체로 부정확)
   - 관련 코드 파일을 전체 읽는다 (생략 없이)
   - 코드 경로를 추적해 실제 근본 원인을 식별한다
   - 수정 방안을 제안한다

3. **기능 요청인 경우**:
   - 관련 코드 파일을 전체 읽는다
   - 가장 간결한 구현 방안을 제안한다
   - 영향받는 파일과 변경사항을 나열한다

## 분석 체크리스트

- API 클라이언트 (`lib/api-client.ts`) 영향 확인
- React Query 훅 (`hooks/query/`) 캐시/무효화 영향 확인
- Zustand 스토어 (`store/`) 상태 영향 확인
- Zod 스키마 검증 영향 확인
- 라우팅 (`app/`) 영향 확인

명시적으로 요청하지 않는 한 구현하지 않는다. 분석과 제안만 수행한다.
